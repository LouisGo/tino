# Tino 日志规范

> 最后更新：2026-04-04

## 1. 目标

- `Rust` 与 `Renderer` 日志都必须落盘
- 日志文件默认写入应用安装体系对应的系统日志目录
- 日志先解决定位问题，不追求复杂平台日志平台

## 2. 当前文件

- `rust.log`：系统监听、归档、批次、文件写入、命令桥
- `renderer.log`：前端运行时、路由、交互异常、未处理 Promise / window error

当前 macOS 路径：

- `~/Library/Logs/com.louistation.tino/rust.log`
- `~/Library/Logs/com.louistation.tino/renderer.log`

## 3. 级别规则

- `debug`：排查阶段的上下文细节，不作为长期业务结论
- `info`：关键生命周期与正常业务节点
- `warn`：可恢复降级，不阻断主链路
- `error`：功能失败、状态异常、需要定位的问题

## 4. 使用规则

- `Rust` 侧统一使用 `log` crate
- `Renderer` 侧统一使用 `src/lib/logger.ts`
- 不再把 `console.*` 作为正式日志接口直接散用
- 新增前端日志时必须带模块语义前缀，例如 `clipboard`、`agent`、`window-close`

## 5. 禁止项

- 不记录完整剪贴板原文
- 不记录 `apiKey`、token、cookie、secret
- 不记录完整 AI 批次 payload

允许记录：

- `capture id`
- `batch id`
- `source app`
- 文件路径
- 字节数、条数、状态、耗时

## 6. 当前滚动策略

- 单文件最大 `10 MB`
- 保留最近 `10` 份归档日志

## 7. 校验

```bash
pnpm check
pnpm tauri dev
```

然后手动检查：

- `rust.log` 是否持续写入
- `renderer.log` 是否在页面报错或 `console.*` 时写入
