# Handoff 扩展归档 2026-04-05

> 目的：把不再适合进入 AI 必读上下文的扩展内容移出 `HANDOFF.md`

## 1. 为什么归档

旧版 `HANDOFF` 同时承担了这些角色：

- 当前状态快照
- 历史阶段说明
- 文件清单
- 详细开发建议
- 新会话提示模板

这会带来两个问题：

- 信息重复，且与其他规划文档形成重叠
- AI 首次阅读时容易把“扩展说明”误当成“当前硬约束”

因此从 `2026-04-05` 起：

- `HANDOFF.md` 只保留当前有效的短版控制信息
- 扩展说明改为归档或专项文档

## 2. 已移出的内容类型

以下内容不再放在 `HANDOFF.md` 主体里：

- 大段文件级清单
- 重复的技术栈展开说明
- 过长的“当前已完成 / 未完成”枚举
- 展开版的下一步推荐理由
- 新会话提示模板

## 4. 历史快照怎么查

如果需要看压缩前的完整 handoff 内容，优先用 git 历史，而不是重新把长文档塞回必读入口。

示例：

```bash
git log -- docs/03-planning/HANDOFF.md
git show <commit>:docs/03-planning/HANDOFF.md
```

## 5. 现在的阅读原则

- 先读短版 `HANDOFF.md`
- 再根据任务读取专项文档
- 不要把归档文档重新提升为默认必读

## 6. 一句结论

归档的目标不是删信息，而是：

> 把“当前必须知道的东西”与“需要时再查的东西”彻底分开。
