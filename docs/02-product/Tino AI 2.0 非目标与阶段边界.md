# Tino AI 2.0 非目标与阶段边界

> 日期：2026-04-26
> 角色：冻结当前阶段故意不做什么，以及为什么不做
> 状态：最新共识

## 1. 为什么需要这份文档

`Tino AI` 的愿景可以很大，但当前阶段如果不把非目标写清楚，产品和架构很容易被未来想象提前绑架。

这份文档的目的就是：

> 明确当前阶段故意不承接哪些职责。

## 2. 当前非目标

### 2.1 不做复杂 Markdown 编辑器

用户可以继续在：

- Notion
- Obsidian
- 思源
- VSCode
- 任意笔记工具

里完成主要编辑工作。

`Tino` 不接管这条主战场。

### 2.2 不做完整笔记软件

`Tino` 当前不是新的笔记 app，也不是新的知识库 UI 系统。

### 2.3 不做完整 workspace 管理系统

当前不追求：

- database-heavy workspace
- 大量协同逻辑
- 权限模型驱动的企业式文档系统

### 2.4 不做通用自动执行 Agent

当前不追求：

- 高自治外部工具执行
- 自动跑复杂任务链
- 把 AI 变成“会做很多事”的黑箱系统

### 2.5 不做陪伴型人格产品

当前不追求：

- 人设一致性
- 长期情感连续性
- 像“朋友”或“伙伴”一样的产品心智

### 2.6 不为某个垂直场景先做重专用系统

例如：

- 小说专用设定宇宙
- 超深研究知识图谱
- 生活管家式主动提醒系统

这些都可能是以后长出来的分支，但不是当前阶段的核心定义。

### 2.7 不做无限延展的闲聊型 chat 线程

当前不把“持续对话本身”定义为核心体验。

每次 chat session 都应尽量绑定：

- 一个 project
- 一个主要任务
- 一个当前 scope

如果没有这些边界，chat 很容易退化成开放式闲聊工具。

## 3. 当前阶段真正允许增长的方向

虽然有很多非目标，但以下增长是允许的，因为它们服务于主战场：

- 更稳的 corpus import / normalize
- 更好的 grounded retrieval
- 更好的 recent digest / resurfacing
- 更好的 Markdown artifact 写回
- 更好的 project template

## 4. 阶段边界

### Phase 2.0 当前阶段

聚焦：

- `project / corpus copilot`
- import / normalize
- grounded ask / research
- draft / distill
- Markdown artifact

### 后续扩展阶段

在当前主循环打穿后，再考虑：

- 更深的创作模板
- 更深的研究模板
- 更深的静默策展能力
- 更复杂的反馈 / 质量闭环
- 更复杂的协作或执行能力

## 5. 一句结论

当前阶段 `Tino AI 2.0` 最重要的克制是：

> 先把“围绕一批材料持续工作”做好，再考虑“围绕整个知识人生持续陪伴”。
