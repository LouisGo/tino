# Tino AI 2.0 结构性差异化

> 日期：2026-04-26
> 角色：冻结 `Tino AI` 相对 `NotebookLM` 与通用云端 AI 产品的结构性差异化
> 状态：最新共识

## 1. 这份文档解决什么问题

当前 2.0 文档已经定义了产品母型、主战场、主循环和运行时契约。

但这些内容还不能直接回答一个更尖锐的问题：

> 为什么用户不直接用 `NotebookLM`，以及为什么即使 `NotebookLM` 继续进化，`Tino` 仍然有自己的路线空间？

这份文档只回答四件事：

- 什么不是 `Tino` 的护城河
- 什么是 `Tino` 的结构性差异
- 为什么这条路线是 `Google` 不容易优先追的
- 这对当前阶段产品优先级意味着什么

## 2. 什么不是护城河

以下内容即使重要，也不应被当成核心护城河：

- 有 `chat`
- 有 `research`
- 支持 PDF / PPT / Markdown / 网页等多格式输入
- 有 grounded answer
- 有引用
- 有摘要和笔记草稿

原因很简单：

这些都是能力差异或实现差异，而不是路线差异。  
`Google` 完全可能继续把这些功能做得更好。

## 3. 真正的结构性差异

`Tino` 当前真正值得坚持的差异，不是“也能研究材料”，而是：

> 它是一个本地优先、系统级接入、面向真实个人工作流的 AI 层。

这条差异可拆成四个部分。

### 3.1 Local-first data plane

`Tino` 默认围绕本地材料、本地目录、本地资产工作。

这里的重点不是“永远不用云”，而是：

- 数据和资产不以云端 workspace 为唯一中心
- 用户的长期材料、草稿、半成品不需要先搬进一个云产品才能工作
- 结果可以继续留在用户自己的工具链中

当前还要再补一层边界：

- `local-first` 当前首先约束的是 data plane 和 workflow plane
- 它不默认等于“Phase 2.0 必须先把本地算力全做完”
- 当前允许显式的 provider-backed cloud inference
- 但不允许把云端推理伪装成“所有东西天然都留在本地”

### 3.2 System-level capture

`Tino` 不只是等用户坐下来“上传一批 sources”。

它的路线是：

- 在真实工作流里接住碎片
- 通过剪贴板、选中内容、文件导入、目录变化等系统级输入通道持续积累

这意味着 `Tino` 的位置不是“研究时才打开的目的地”，而是“工作持续发生时一直在旁边接住材料的一层”。

### 3.3 Open artifact loop

`Tino` 的结果不是困在对话记录里，也不是困在某个封闭 workspace 里。

它的路线是：

- 结果变成可带走的 Markdown artifact
- 用户可以在 `Obsidian / 思源 / VSCode / 其他本地工具` 里继续编辑
- 外部修改结果还能重新回流成新的 revision / corpus signal

这是一条开放闭环，而不是单产品黑箱闭环。

### 3.4 Project re-entry

`Tino` 不只是回答某次问题，而是帮助用户重新进入一个正在推进的 project。

也就是说，它的价值不仅是：

- “我现在研究一下这批材料”

更是：

- “我昨天做到哪了”
- “我今天继续推进哪个 project”
- “我最近积累的碎片和上次产出的 memo 能不能接起来”

## 4. 为什么这是 `Google` 不容易优先追的路线

这里说的不是“做不到”，而是“不容易优先做深”。

`Google` 这类云产品天然更倾向于：

- 云端为中心
- 通用体验
- 网页化
- 跨平台一致性
- 低系统耦合

而 `Tino` 当前路线刻意站在另一边：

- 更本地
- 更贴近 `macOS` 系统能力
- 更深接入个人文件与输入工作流
- 更强调开放 artifact，而不是把结果留在产品内

这类路线不是不能被模仿，但它会和通用网页产品的默认优化方向天然冲突。

## 5. 这对当前产品优先级意味着什么

如果以上判断成立，那么当前阶段优先级就应被重新解释为：

### 5.1 该优先做强的

- 剪贴板与系统级 capture
- import / normalize / project routing
- grounded ask / research
- project re-entry
- Markdown artifact 输出与回流

### 5.2 不该误判成护城河的

- 表面上更多的 AI 功能按钮
- 更像编辑器
- 更像通用云 workspace
- 更像“什么都能问”的万能 chat

## 6. 一句结论

`Tino` 的护城河不是“也能研究材料”，而是：

> 它是一个本地优先、系统级接入、开放 artifact 闭环的个人工作流 AI 层。`NotebookLM` 帮你研究材料；`Tino` 帮你在日常工作中持续接住材料、重新进入 project，并把结果沉淀回你自己的工具链。
