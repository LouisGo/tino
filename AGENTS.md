## Engineering Principles

- DRY at app scope: when similar logic appears across modules, extract a shared `command`, pure helper, or service instead of duplicating local variants.
- Logic before UI: separate pure or domain logic from React and UI state first; keep heavy or reusable logic out of React so UI code stays thin.
- Composition over inheritance.
- Separate concerns: keep related logic close together, and split unrelated responsibilities.
- Prefer high cohesion and low coupling.

## Rust IPC Flow

Use this whenever a renderer-facing type crosses the Rust <-> renderer boundary.

1. Define or update the type in Rust.
2. Add `specta::Type` to the Rust type derive list.
3. If the type is used by a Tauri command, annotate the command with `#[specta::specta]`.
4. Register new commands in `src-tauri/src/ipc_schema.rs`.
5. Run `pnpm gen:bindings`.
6. Only after bindings are generated, write or update renderer code that consumes the new type.

## Rust-Owned Types

These must originate in Rust and flow through generated bindings:

- Tauri command request types
- Tauri command response types
- Tauri event payload types
- Shared DTOs persisted or enforced by the Rust runtime

Do not hand-write matching renderer copies for these shapes.

## Renderer-Only Types

These can stay in TypeScript:

- Local component props
- UI state models
- View-model types derived from Rust data
- Temporary form-only helper types
- Pure renderer utility types with no Tauri IPC involvement

## Renderer Consumption

- Generated bindings live in `src/bindings/tauri.ts`. Do not edit this file manually.
- Renderer code should consume generated command bindings through `src/lib/tauri.ts`.
- Renderer code should consume generated and shared type aliases through `src/types/shell.ts`.

## Tauri Commands

- Use the repo wrapper commands instead of `pnpm exec tauri ...`.
- `pnpm tauri dev` or `pnpm tauri:dev`: development env on the shared preview data channel.
- `pnpm tauri build` or `pnpm tauri:build:test`: staging and test bundle on the shared preview data channel.
- `pnpm tauri build --production` or `pnpm tauri:build:prod`: production bundle on the isolated production data channel.
- `pnpm tauri:build-install:test` / `pnpm tauri:build-install:staging`: build and install the preview app into `/Applications`.
- `pnpm tauri:build-install:prod`: build and install the production app into `/Applications`.
- On macOS, run `pnpm macos:setup-local-signing` before validating Accessibility-dependent flows such as clipboard paste-back; ad-hoc signing is not a stable substitute for permission testing.
- Avoid `pnpm exec tauri ...` unless you are intentionally debugging the raw Tauri CLI.
- Before `pnpm tauri dev`, `pnpm build`, and `pnpm typecheck`, bindings are generated automatically.
- If you changed Rust IPC shapes in the current task, run `pnpm gen:bindings` before finishing even if no one explicitly asked.
